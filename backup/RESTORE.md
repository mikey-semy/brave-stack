# Восстановление / переезд на новый сервер

Принцип «системы ухода»: всё ценное лежит в **S3/B2-бэкапах вне этого сервера**,
а инфраструктура — в этом репозитории. Переезд = clone + распаковка data + `up`.

## Что нужно для восстановления
- Доступ к бакету (те же ключи, что в `backup/backup.env`).
- GPG-приватный ключ, **если** бэкапы шифровались (`GPG_RECIPIENT`). Без него
  зашифрованный архив не открыть — храни этот ключ отдельно и надёжно!

## Шаги переезда
1. Новый сервер (Linux + Docker + compose), клонируй репозиторий:
   ```bash
   git clone https://github.com/mikey-semy/remote-app.git && cd remote-app
   ```
2. Настрой rclone-доступ (скопируй старый `backup/backup.env` или заполни заново
   и запусти `bash backup/setup-backup.sh` — он поставит rclone и настроит remote).
3. Найди и скачай последний бэкап:
   ```bash
   rclone lsf backup:ИМЯ_БАКЕТА/brave        # список архивов
   rclone copy backup:ИМЯ_БАКЕТА/brave/<последний> ./restore/
   ```
4. Если архив зашифрован — расшифруй:
   ```bash
   gpg --decrypt restore/<имя>.tar.gz.gpg > restore/<имя>.tar.gz
   ```
5. Распакуй в корень (пути в архиве абсолютные):
   ```bash
   sudo tar xzf restore/<имя>.tar.gz -C /
   ```
6. Подними сервисы:
   ```bash
   echo "RELAY_HOST=<НОВЫЙ_IP>" > .env
   docker compose up -d
   ```
   > RustDesk `data/` восстановлен → **ключ тот же**. Клиентам нужно поменять
   > только **Server ID** (новый IP), поле **Key оставить прежним**.
7. Обнови IP в клиентах (и DNS, если используешь домен).

## Регулярная проверка (важно!)
Бэкап без проверки восстановления — это не бэкап. Раз в пару месяцев:
```bash
rclone lsl backup:ИМЯ_БАКЕТА/brave     # свежие даты и размеры на месте?
```
И хотя бы раз сделай тестовую распаковку в `/tmp`, чтобы убедиться, что архив целый
(и GPG-ключ на месте).
